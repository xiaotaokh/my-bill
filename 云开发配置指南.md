# 云开发环境配置说明

## 第一步：创建云开发环境

1. 打开微信开发者工具
2. 点击顶部菜单「云开发」
3. 如果是首次使用，会弹出「开通云开发」对话框
4. 点击「开通」，选择「按量付费」或「按量付费（可设置免费资源包）」
5. 记录生成的环境ID（格式类似：cloud1-xxx 或 production-xxx）

## 第二步：配置环境ID

### 方法一：在 app.js 中配置

打开 `app.js`，修改第6行的环境ID：

```javascript
wx.cloud.init({
  env: 'production-xxx', // ← 将这里替换为你的环境ID
  traceUser: true,
})
```

### 方法二：在 project.config.json 中配置（推荐）

1. 打开微信开发者工具
2. 点击右上角「详情」
3. 选择「本地设置」
4. 找到「云开发环境」配置项
5. 选择你创建的环境

## 第三步：初始化数据库

1. 在微信开发者工具中点击「云开发」
2. 进入云开发控制台
3. 点击左侧菜单「数据库」
4. 点击「+」创建集合
5. 集合名称输入：`assets`
6. 点击「确定」

### 设置数据库权限

1. 在数据库页面，找到 `assets` 集合
2. 点击集合名称进入详情
3. 点击「权限设置」
4. 选择「仅创建者可读写」
5. 点击「保存」

这样设置后，每个用户只能访问和修改自己的数据。

## 第四步：部署云函数

### 批量部署（推荐）

1. 在微信开发者工具中，展开 `cloudfunctions` 目录
2. 右键点击第一个云函数文件夹（如 `addAsset`）
3. 选择「上传并部署：云端安装依赖」
4. 等待部署完成
5. 重复步骤2-4，部署所有云函数（`getAssets`, `getStats`, `updateAsset`, `deleteAsset`）

### 验证部署

1. 部署完成后，云函数文件夹图标会变成「云朵」形状
2. 在云开发控制台的「云函数」页面可以看到已部署的函数

## 第五步：运行项目

1. 点击微信开发者工具的「编译」按钮
2. 等待编译完成
3. 在模拟器中查看效果
4. 点击「预览」或「真机调试」在手机上测试

## 常见问题

### Q1: 云函数部署失败？

**可能原因1**: 未开通云开发
- 解决：先按照第一步开通云开发

**可能原因2**: 网络问题
- 解决：检查网络连接，重试部署

**可能原因3**: 环境未选择
- 解决：在项目配置中选择正确的云环境

### Q2: 数据库操作失败？

**可能原因**: 数据库权限设置不正确
- 解决：检查数据库权限是否设置为「仅创建者可读写」

### Q3: 环境ID在哪里查看？

在云开发控制台的右上角可以看到当前环境的ID。

### Q4: 如何查看日志？

1. 在云开发控制台
2. 点击「云函数」
3. 选择要查看的云函数
4. 点击「日志」标签页
