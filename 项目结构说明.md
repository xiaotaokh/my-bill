# 项目文件结构说明

## 根目录文件

```
myBill/
├── app.js                      # 小程序入口文件，初始化云开发
├── app.json                    # 小程序全局配置（页面路由、窗口样式等）
├── app.wxss                    # 全局样式文件
├── project.config.json         # 项目配置（AppID、云函数路径等）
├── sitemap.json                # 小程序索引配置
├── CLAUDE.md                   # Claude Code 开发指南
├── README.md                   # 项目说明文档
├── 需求文档.md                  # 详细需求文档
├── 云开发配置指南.md             # 云开发环境配置说明
├── install.sh                  # 云函数依赖安装脚本
└── 首页.jpg                     # 首页设计稿参考
```

## 页面目录 (pages/)

### 1. 首页 (pages/index/)
```
index/
├── index.js        # 页面逻辑：加载数据、筛选、排序、统计计算
├── index.wxml      # 页面结构：总览卡片、筛选栏、排序栏、资产列表
├── index.wxss      # 页面样式：渐变背景、卡片、列表等
└── index.json      # 页面配置：标题、下拉刷新
```

**功能说明**:
- 资产总览统计（总价、日均成本、各状态数量）
- 状态筛选（全部/服役中/已退役/已卖出）
- 多维度排序（价格、购买时间、服役时长、日均成本）
- 资产列表展示
- 添加资产按钮（浮动按钮）

### 2. 添加资产页 (pages/asset-add/)
```
asset-add/
├── asset-add.js        # 表单逻辑：验证、提交、类别管理
├── asset-add.wxml      # 表单结构：名称、价格、日期、类别、备注、开关选项
├── asset-add.wxss      # 表单样式：输入框、选择器、开关等
└── asset-add.json      # 页面配置：标题栏颜色
```

**功能说明**:
- 表单填写（物品名称、价格、购买日期、类别、备注）
- 类别选择和自定义添加
- 状态设置（已退役/已卖出）
- 特殊选项（不计入总资产/不计入日均）
- 表单验证
- 数据保存到云数据库

### 3. 资产详情页 (pages/asset-detail/)
```
asset-detail/
├── asset-detail.js        # 详情逻辑：加载、编辑、删除
├── asset-detail.wxml      # 详情结构：状态标签、基本信息、操作按钮
├── asset-detail.wxss      # 详情样式：卡片、信息项、按钮
└── asset-detail.json      # 页面配置：标题栏颜色
```

**功能说明**:
- 显示资产完整信息
- 计算服役时长和日均成本
- 编辑资产（跳转到编辑模式的添加页）
- 删除资产（带确认提示）

## 文件关系图

```
用户打开小程序
    ↓
app.js (初始化)
    ↓
pages/index/index.js (首页加载)
    ├── 云函数: getAssets (获取列表)
    ├── 云函数: getStats (获取统计)
    └── 显示页面
         ├── 点击 [+]
         │    └── 跳转到 asset-add (添加资产)
         │         └── 云函数: addAsset (保存)
         │
         └── 点击资产卡片
              └── 跳转到 asset-detail (查看详情)
                   ├── 编辑 → 跳转到 asset-add (编辑模式)
                   │    └── 云函数: updateAsset (更新)
                   │
                   └── 删除
                        └── 云函数: deleteAsset (删除)
```

## 开发流程

1. **配置环境**
   - 配置AppID
   - 配置云开发环境
   - 初始化数据库

2. **部署云函数**
   - 上传所有云函数
   - 验证部署成功

3. **开发调试**
   - 在开发者工具中修改代码
   - 实时预览效果
   - 查看控制台日志

4. **测试功能**
   - 添加资产
   - 查看列表
   - 筛选排序
   - 编辑删除

5. **发布上线**
   - 提交审核
   - 发布版本

## 注意事项

1. **云函数需要部署**：本地修改云函数代码后，需要重新上传部署
2. **数据库权限**：确保设置为「仅创建者可读写」
3. **环境ID**：开发和生产环境使用不同的环境ID
4. **依赖安装**：云函数修改依赖后需要重新安装和部署
