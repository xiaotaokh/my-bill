# 📋 项目部署检查清单

使用前请按顺序检查以下项目，确保所有配置正确。

## ✅ 环境准备

- [ ] 已安装微信开发者工具（最新版本）
- [ ] 已使用微信账号登录开发者工具
- [ ] 已导入项目到开发者工具

## ✅ 云开发配置

- [ ] 已点击「云开发」按钮
- [ ] 已开通云开发环境
- [ ] 已记下环境ID（格式：production-xxx）
- [ ] 已在项目配置中设置环境（「详情」→「本地设置」→「云开发环境」）

## ✅ 数据库初始化

- [ ] 已进入云开发控制台
- [ ] 已在「数据库」中创建 `assets` 集合
- [ ] 已设置数据库权限为「仅创建者可读写」

## ✅ 云函数部署

逐个检查以下云函数是否部署成功（文件夹图标应变为云朵形状）：

- [ ] `addAsset` - 已部署
- [ ] `getAssets` - 已部署
- [ ] `getStats` - 已部署
- [ ] `updateAsset` - 已部署
- [ ] `deleteAsset` - 已部署

**部署方法**：
1. 右键点击云函数文件夹
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成

## ✅ 代码配置

- [ ] `app.js` 中的 `env` 已配置为你的环境ID
- [ ] `app.js` 中的 `globalData.envId` 已配置为你的环境ID

**配置位置**：`app.js` 第6行和第29行

```javascript
// 第6行
wx.cloud.init({
  env: '你的环境ID', // ← 这里
  traceUser: true,
})

// 第29行
globalData: {
  userInfo: null,
  envId: '你的环境ID' // ← 这里
}
```

## ✅ 常见问题排查

### 如果遇到问题，按以下顺序排查：

1. **云函数调用失败**
   - 检查：云函数是否已部署？
   - 检查：环境是否选择正确？
   - 查看：云函数日志（云开发控制台 → 云函数 → 日志）

2. **数据库操作失败**
   - 检查：数据库权限是否正确？
   - 检查：集合名称是否正确（`assets`）？
   - 查看：数据库操作日志

3. **页面显示异常**
   - 检查：编译是否有错误？
   - 查看：开发者工具控制台是否有报错？
   - 检查：数据是否正确加载？

4. **统计信息不准确**
   - 检查：数据是否包含 `excludeTotal` 或 `excludeDaily` 标记？
   - 检查：状态字段是否正确（active/retired/sold）？
