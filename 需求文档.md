# 微信小程序 - 个人资产统计

## 1. 项目概述

### 1.1 项目名称
我的资产（myBill）

### 1.2 项目定位
个人资产管理工具，帮助用户统计和追踪名下各类资产。

### 1.3 技术方案
- **前端**: 微信小程序原生开发
- **后端**: 微信云开发（CloudBase）
- **数据库**: 云开发数据库
- **登录**: 微信账号授权登录
- **存储**: 云存储（如需存储图片等）

---

## 2. 用户需求

### 2.1 用户故事
1. 作为用户，我希望能够快速添加我的资产信息，以便追踪我的资产情况
2. 作为用户，我希望能够查看我的总资产价值，以便了解我的财务状况
3. 作为用户，我希望能够按状态分类查看资产，以便管理不同状态的资产
4. 作为用户，我希望能够对资产进行多维度排序，以便快速定位关键资产

---

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 首页（tabBar）
- **添加资产按钮**: 快捷入口，点击后跳转到添加资产页面
- **资产总览卡片**: 显示关键统计数据

#### 3.1.2 资产总览
显示以下统计信息：
- **总资产价格**: 所有资产的当前价值总和
- **日均成本**: 总成本 / 服役总天数
- **服役中个数**: 状态为"服役中"的资产数量
- **已退役个数**: 状态为"已退役"的资产数量
- **已卖出个数**: 状态为"已卖出"的资产数量

#### 3.1.3 资产列表
- **筛选功能**: 按状态筛选（全部/服役中/已退役/已卖出）
- **排序功能**: 支持按以下维度排序
  - 价格（升序/降序）
  - 添加时间（升序/降序）
  - 购买时间（升序/降序）
  - 服役时长（升序/降序）
  - 日均成本（升序/降序）
- **列表展示**: 每个资产卡片显示关键信息
  - 资产名称
  - 价格
  - 状态标签
  - 购买时间
  - 日均成本

#### 3.1.4 添加资产
- **表单字段**:
  - 资产名称（必填）
  - 资产分类（下拉选择：电子设备、房产、车辆、投资、其他等）
  - 价格/购买金额（必填，数字）
  - 购买时间（必填，日期选择器）
  - 状态（默认"服役中"，下拉选择）
  - 备注（可选，文本域）
- **提交保存**: 保存到云数据库

#### 3.1.5 资产详情
- **查看详细信息**: 展示资产的完整信息
- **编辑功能**: 修改资产信息
- **删除功能**: 删除资产（二次确认）

## 4. 数据模型

### 4.1 资产表（assets）

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| _id | String | 数据库自动生成 | ✓ |
| _openid | String | 微信用户唯一标识 | ✓ |
| name | String | 资产名称 | ✓ |
| category | String | 资产分类（电子设备/房产/车辆/投资/其他） | ✓ |
| price | Number | 购买价格 | ✓ |
| purchaseDate | Date | 购买日期 | ✓ |
| status | String | 状态（active/retired/sold） | ✓ |
| remark | String | 备注 | ✗ |
| createdAt | Timestamp | 创建时间 | ✓ |
| updatedAt | Timestamp | 更新时间 | ✓ |

### 4.2 计算字段

- **服役时长**: 当前日期 - 购买日期（仅状态为"服役中"时计算）
- **日均成本**: 价格 / 服役天数

## 7. API接口（云函数）

### 7.1 云函数列表
- `addAsset`: 添加资产

- `getAssetStats`: 获取资产统计信息
- `updateAsset`: 更新资产信息
- `deleteAsset`: 删除资产
