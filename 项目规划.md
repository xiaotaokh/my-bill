# 资产管理微信小程序需求文档（PRD）

## 1. 项目概述

### 1.1 项目名称
资产管理小程序（暂定）

### 1.2 项目目标
开发一个用于记录和统计个人资产的小程序，用户可以记录名下资产，并查看资产统计数据、分类列表以及排序信息。

### 1.3 技术方案

前端：微信小程序原生开发  
后端：微信云开发（Serverless）  
数据库：云数据库  
用户系统：使用微信 `openid` 自动识别用户

系统架构：

```
小程序前端
    │
    │ wx.cloud API
    │
微信云开发
    │
云数据库
```

---

# 2. 用户系统

## 2.1 登录方式

用户进入小程序后自动登录。

流程：

```
用户打开小程序
↓
获取 openid
↓
自动建立用户数据
```

用户唯一标识：

```
openid
```

所有资产数据与 `openid` 绑定。

# 5. 添加资产

用户可以新增资产记录。

## 5.1 输入字段

| 字段 | 类型 | 是否必填 | 说明 |
|-----|-----|-----|-----|
| 资产名称 | string | 是 | 资产名称 |
| 资产状态 | enum | 是 | 服役中 / 已退役 / 已卖出 |
| 当前价格 | number | 是 | 当前资产价格 |
| 购买价格 | number | 否 | 资产购买价格 |
| 购买时间 | date | 否 | 购买日期 |
| 添加时间 | datetime | 自动 | 创建时间 |
| 备注 | string | 否 | 备注说明 |

---

# 6. 自动计算字段

系统自动计算以下信息：

| 字段 | 计算方式 |
|-----|-----|
| 服役时长 | 当前时间 - 购买时间 |
| 日均成本 | 购买价格 ÷ 服役天数 |

说明：

```
服役天数 = 当前日期 - 购买日期
```

单位：天

---

# 7. 资产总览

用于展示当前用户的资产整体情况。

统计范围：

```
当前用户 openid
```

## 7.1 展示数据

| 统计项 | 说明 |
|-----|-----|
| 总资产价格 | 所有资产价格总和 |
| 日均成本 | 所有资产日均成本平均值 |
| 服役中个数 | 状态=服役中 |
| 已退役个数 | 状态=已退役 |
| 已卖出个数 | 状态=已卖出 |

# 11. 数据库设计

数据库集合：

```
assets
```

字段结构：

| 字段 | 类型 | 说明 |
|-----|-----|-----|
| _id | string | 数据ID |
| _openid | string | 用户ID |
| name | string | 资产名称 |
| status | string | 资产状态 |
| price | number | 当前价格 |
| buyPrice | number | 购买价格 |
| buyDate | date | 购买日期 |
| createTime | datetime | 添加时间 |
| remark | string | 备注 |

---

# 12. 统计逻辑

## 12.1 总资产价格

```
sum(price)
```

---

## 12.2 日均成本

单资产计算：

```
日均成本 = 购买价格 ÷ 服役天数
```

整体统计：

```
平均(日均成本)
```

---

## 12.3 服役时长

计算方式：

```
服役时长 = 当前日期 - 购买日期
```

单位：天
