# Feature Specification: 订阅类资产 (Subscription Asset)

**Feature Branch**: `001-subscription-asset`
**Created**: 2026-03-27
**Status**: Draft
**Input**: 用户需求：支持订阅类资产（如音乐会员），每期固定支出，总投入动态累计，日均成本固定

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 新增订阅资产 (Priority: P1)

用户希望记录一个订阅类资产（如音乐会员），首先选择资产类型为"订阅"，然后设置订阅周期和每期金额，系统自动计算动态累计的总投入和固定的日均成本。

**Why this priority**: 这是核心功能，没有这个功能，用户无法创建订阅类资产。

**Independent Test**: 用户可以在新增资产页面先选择"订阅"类型，填写名称、购买日期、每期金额、周期等信息，保存后能在资产列表中看到带有"订阅"标签的资产，显示正确的日均成本。

**Acceptance Scenarios**:

1. **Given** 用户在新增资产页面，**When** 选择资产类型为"订阅"，输入名称"音乐会员"、每期金额88元、周期为"月度"、购买日期为2026-01-01（待订阅开关关闭），**Then** 资产保存成功，状态为"服役中"，日均成本显示为 88÷30 ≈ 2.93元/天
2. **Given** 用户在新增资产页面，**When** 选择资产类型为"订阅"，打开"待订阅"开关，选择开始日期为2026-04-01，**Then** 资产保存成功，状态为"待生效"，资产列表显示"待生效"标签
3. **Given** 用户已创建月度订阅资产，**When** 查看资产详情，**Then** 总投入显示为每期金额 × 已过周期数，日均成本固定不变
4. **Given** 用户选择自定义周期，**When** 输入周期天数为90天（季度），每期金额为268元，**Then** 日均成本显示为 268÷90 ≈ 2.98元/天

---

### User Story 2 - 编辑订阅资产 (Priority: P2)

用户可以修改已有的订阅资产信息，包括调整每期金额、变更订阅周期、设置结束日期或手动结束订阅。

**Why this priority**: 用户可能需要调整订阅金额（如会员涨价）或结束订阅。

**Independent Test**: 用户可以进入编辑页面修改订阅资产信息，保存后数据正确更新。

**Acceptance Scenarios**:

1. **Given** 用户有一个月度订阅资产，**When** 编辑并将每期金额从88元改为98元，**Then** 保存后日均成本更新为 98÷30 ≈ 3.27元/天，总投入分段计算（修改前按88元，修改后按98元）
2. **Given** 用户有一个订阅资产，**When** 设置结束日期为2026-06-30，**Then** 到期后资产自动标记为已结束，不再计入日均统计
3. **Given** 用户有一个订阅资产，**When** 点击"结束订阅"按钮，**Then** 资产立即标记为已结束，结束日期为当天

---

### User Story 3 - 订阅资产统计展示 (Priority: P2)

用户在首页可以看到订阅类资产的统计数据，包括动态累计的总投入和日均成本，与其他资产正确合并显示。

**Why this priority**: 用户需要直观看到订阅资产对总体财务的影响。

**Independent Test**: 首页统计数据正确包含订阅类资产，显示合理的总资产和日均成本。

**Acceptance Scenarios**:

1. **Given** 用户有订阅资产和普通资产，**When** 查看首页统计，**Then** 总资产 = 普通资产价格之和 + 订阅资产动态累计总额，日均成本 = 各资产日均之和
2. **Given** 用户有已结束的订阅资产，**When** 查看首页统计，**Then** 已结束的订阅资产不计入当前日均，但累计总投入仍计入历史统计
3. **Given** 用户有订阅资产且开启了"不计入总日均"，**When** 查看首页统计，**Then** 该订阅资产的日均成本不计入总日均

---

### Edge Cases

- 订阅开始日期在未来时 → 正常显示在列表中，用"待生效"标签标记，不计入统计
- 自定义周期输入0或负数时，如何处理？（校验拒绝）
- 订阅跨越多年时，累计周期数是否正确？（按实际天数计算）
- 用户修改开始日期后，累计总投入是否重新计算？（是，动态计算）

## Clarifications

### Session 2026-03-27

- Q: 当用户修改订阅资产的每期金额时，历史累计的总投入应该如何处理？ → A: 保留历史记录，仅从修改之日起按新金额计算
- Q: 当订阅开始日期在未来时，该资产在资产列表中如何显示？ → A: 正常显示在列表中，用标签标记"待生效"
- Q: 订阅资产的完整新增流程和字段设计？ → A: 新增资产时先选择类型（普通/订阅），订阅资产字段包括名称、购买日期、缩略图、分类、每期金额、周期类型、周期天数、结束日期、订阅状态、备注、其他选项（待订阅开关/不计入总资产/不计入总日均）。资产列表用"订阅"标签区分，待订阅开关打开后资产状态为"待生效"，默认为"服役中"

## Requirements *(mandatory)*

### Functional Requirements

**资产类型选择**
- **FR-001**: 系统 MUST 支持新增资产时先选择资产类型：普通资产 / 订阅资产
- **FR-002**: 系统 MUST 保持普通资产的所有现有功能完全不变

**订阅资产字段**
- **FR-003**: 系统 MUST 支持订阅资产的以下必填字段：名称、购买日期、每期金额、周期类型
- **FR-004**: 系统 MUST 支持订阅资产的以下可选字段：缩略图、分类、周期天数（自定义时）、结束日期、备注
- **FR-005**: 系统 MUST 支持订阅周期类型：月度（30天）、年度（365天）、周（7天）、自定义天数

**订阅资产选项**
- **FR-006**: 系统 MUST 提供"待订阅"开关：关闭时资产状态为"服役中"，打开时显示开始日期选择器，资产状态为"待生效"
- **FR-007**: 系统 MUST 支持"不计入总资产"选项
- **FR-008**: 系统 MUST 支持"不计入总日均"选项

**计算逻辑**
- **FR-009**: 系统 MUST 动态计算订阅资产的总投入 = 每期金额 × 已过周期数（向下取整）
- **FR-009a**: 当用户修改每期金额时，系统 MUST 保留历史记录，仅从修改之日起按新金额计算，分段累计总投入
- **FR-010**: 系统 MUST 计算订阅资产的日均成本 = 每期金额 ÷ 周期天数（固定值）

**订阅生命周期**
- **FR-011**: 系统 MUST 支持用户设置订阅结束日期，到期后自动停止累计
- **FR-012**: 系统 MUST 支持用户手动结束订阅，立即停止累计

**列表展示**
- **FR-013**: 系统 MUST 在资产列表中用"订阅"标签区分订阅资产和普通资产
- **FR-014**: 系统 MUST 在资产列表中用"待生效"标签标记未来开始的订阅资产

### Key Entities

- **Asset（资产）**: 新增 `assetType` 字段区分资产类型
  - `assetType`: 资产类型，'fixed'（普通资产，默认值）或 'subscription'（订阅资产）

- **SubscriptionAsset（订阅资产扩展字段）**: 当 assetType='subscription' 时使用
  - `periodAmount`: 每期金额（当前值）
  - `periodType`: 周期类型，'monthly' | 'yearly' | 'weekly' | 'custom'
  - `periodDays`: 周期天数（periodType为custom时使用，默认根据periodType计算）
  - `subscriptionStartDate`: 订阅开始日期（待订阅开关打开时使用）
  - `subscriptionEndDate`: 订阅结束日期（可选）
  - `subscriptionStatus`: 订阅状态，'pending'（待生效）| 'active'（服役中）| 'ended'（已结束）
  - `amountHistory`: 金额变更历史数组，每项包含 `{amount, effectiveDate}` 用于分段计算总投入
  - `pendingSubscription`: 待订阅开关，boolean，true时显示开始日期选择器

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可以在3步内完成订阅资产的新增（选择类型 → 设置订阅参数 → 保存）
- **SC-002**: 订阅资产的日均成本显示准确，误差不超过0.01元
- **SC-003**: 订阅资产的总投入动态更新准确，用户无需手动刷新
- **SC-004**: 现有资产功能不受影响，所有已有数据正常显示
- **SC-005**: 用户可以在资产列表中区分订阅资产和普通资产

## Assumptions

- 月度周期按30天计算，年度按365天计算（不考虑闰年差异）
- 订阅周期从开始日期的次日开始计算（购买当天算第1天）
- 自定义周期支持1-365天范围内的任意天数
- 订阅资产的"购买日期"对应"订阅开始日期"
- 已结束的订阅资产保留历史累计数据，可查看但不再计入当前统计
