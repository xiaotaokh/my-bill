# Feature Specification: 时间段资产统计图

**Feature Branch**: `001-asset-time-chart`
**Created**: 2026-03-18
**Status**: Draft
**Input**: User description: "统计页面添加一个统计图,可以查看各个时间段的资产信息"

## Clarifications

### Session 2026-03-18

- Q: 统计图应使用哪种图表类型？ → A: 柱状图 - 显示各时间段总金额，适合对比
- Q: "全部"时间段的图表分组方式？ → A: 支持按年/按月/按季度三种时间线统计，默认按年分组
- Q: 统计图在统计页面的位置？ → A: 放在最下方，位于折线图之后
- Q: 柱状图展示的数据维度？ → A: 单柱展示金额 + Y轴双刻度，左侧金额、右侧数量
- Q: 时间段选择是否影响其他图表？ → A: 独立展示，时间段统计图独立运作，不影响环形图和折线图

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看时间段资产统计 (Priority: P1)

用户在统计页面选择一个时间段（如本周、本月、本年），系统展示该时间段内购买的资产总金额和资产数量，帮助用户了解特定时间范围内的资产消费情况。

**Why this priority**: 这是核心功能，用户最关心的是能够快速查看不同时间维度的资产统计，这是该功能的主要价值所在。

**Independent Test**: 可以通过切换时间段选择器，观察统计图数据的变化来独立测试，验证不同时间段的数据准确性。

**Acceptance Scenarios**:

1. **Given** 用户进入统计页面，**When** 用户看到时间段资产统计模块，**Then** 默认显示"本月"的资产统计数据
2. **Given** 用户正在查看时间段统计，**When** 用户切换到"本年"时间段，**Then** 统计图更新展示本年所有购买资产的金额汇总
3. **Given** 用户选择"本周"时间段，**When** 本周内无资产购买记录，**Then** 显示友好的空状态提示

---

### User Story 2 - 时间段快速切换 (Priority: P2)

用户可以通过预设的时间段选项（本周、本月、本季度、本年、全部）快速切换查看不同维度的资产统计，无需手动输入日期范围。

**Why this priority**: 快速切换功能提升用户体验，让用户能够便捷地在不同时间维度间对比分析。

**Independent Test**: 点击每个时间段选项，验证切换是否流畅、数据是否正确更新。

**Acceptance Scenarios**:

1. **Given** 用户查看时间段统计，**When** 用户点击"本季度"选项，**Then** 统计图立即更新为本季度的数据
2. **Given** 用户切换时间段，**When** 切换完成，**Then** 当前选中的时间段选项有明确的视觉高亮标识

---

### User Story 3 - 资产分类时间统计 (Priority: P3)

在选定的时间段内，用户可以看到各分类资产在该时间段内的购买金额分布，帮助用户分析特定时期的消费偏好。

**Why this priority**: 这是对核心功能的增强，提供更细粒度的分析能力，但用户主要需求是总体统计。

**Independent Test**: 选择不同时间段，验证分类数据是否随时间段正确筛选。

**Acceptance Scenarios**:

1. **Given** 用户选择"本月"时间段，**When** 查看分类分布，**Then** 仅显示本月内购买资产的分类统计
2. **Given** 某分类在选定时间段内无资产，**When** 显示分类列表，**Then** 该分类不显示或显示为0

---

### Edge Cases

- 用户首次使用，尚无任何资产时，时间段统计显示空状态提示
- 时间段跨越多个年份时（如"全部"），支持按年/按月/按季度三种分组方式，默认按年分组展示
- 资产购买日期缺失时，不计入时间段统计
- 用户设备时间异常时，以服务器时间为准计算"本周/本月"

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在统计页面展示时间段资产统计模块
- **FR-002**: 系统必须提供时间段选择器，包含预设选项：本周、本月、本季度、本年、全部
- **FR-003**: 系统必须根据用户选择的时间段，筛选并统计该时间段内购买资产的总金额
- **FR-004**: 系统必须展示选定时间段内的资产购买数量
- **FR-005**: 系统必须使用柱状图可视化展示时间段统计数据，采用Y轴双刻度设计：左侧Y轴显示金额（单位：元），右侧Y轴显示数量（单位：件），柱子高度表示资产总金额
- **FR-008**: 当选择"全部"时间段时，系统必须提供时间线粒度切换选项：按年、按月、按季度，默认为按年分组
- **FR-009**: 时间段资产统计模块必须位于统计页面最下方，在现有折线图之后
- **FR-010**: 时间段统计图必须独立运作，时间段选择不影响页面其他图表（环形图、折线图）的数据展示
- **FR-006**: 用户切换时间段时，统计图必须在1秒内完成数据更新
- **FR-007**: 系统必须显示每个时间段选项的选中状态

### Key Entities

- **时间段选项**: 包含选项名称（本周/本月/本季度/本年/全部）、对应的日期范围计算逻辑
- **时间统计数据**: 时间段标识、资产总金额、资产数量、各分类金额分布

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户能够在3秒内找到并理解时间段统计模块
- **SC-002**: 时间段切换后，数据在1秒内完成更新展示
- **SC-003**: 统计数据与实际资产购买日期完全匹配，准确率100%
- **SC-004**: 用户能够在无需指导的情况下完成时间段切换操作
- **SC-005**: 空数据状态下，用户能看到清晰的提示信息

## Assumptions

- "本周"定义为周一至周日
- "本月"定义为当月1日至当月最后一天
- "本季度"定义为当前所在的自然季度
- "本年"定义为当年1月1日至12月31日
- 资产的"购买时间"以 purchaseDate 字段为准
- 统计仅计算资产价格(price)，不包括其他费用
- 图表使用项目现有的 echarts 组件实现
